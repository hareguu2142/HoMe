Git Commit History 삭제 방법

Git commit history 모두 삭제하기

방법

Git commit history를 완전히 삭제하려면 다음 단계를 따라야 합니다. 이 방법은 모든 커밋 기록을 제거하고 새로 시작하는 방식이므로 주의해서 사용해야 합니다.

  1. 기존 커밋 이력 삭제: 
    
# 현재 브랜치의 모든 커밋 이력을 삭제합니다.
git checkout --orphan latest_branch  # 새로운 브랜치를 생성하고 이동합니다.
git add -A  # 모든 파일을 스테이징합니다.
git commit -am "Initial commit"  # 새로운 첫 커밋을 만듭니다.
  2. 브랜치 이름 변경: 
    
git branch -D main  # 기존 브랜치를 삭제합니다. (예: main, master 등)
git branch -m main  # 현재 브랜치를 기존 브랜치 이름으로 변경합니다.
  3. 원격 저장소에 강제로 푸시: 
    
git push -f origin main  # 모든 이력을 삭제하고 원격 저장소에 강제로 푸시합니다.

이 과정을 완료하면 원격 저장소와 로컬 저장소 모두에서 이전 커밋 이력이 사라지고 새 커밋 이력만 남게 됩니다.

주의: 이 작업은 되돌릴 수 없으므로, 작업 전에 백업을 만들어 두는 것이 좋습니다.
