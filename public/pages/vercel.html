Vercel 설정 정보

이 페이지는 Vercel 배포 설정에 대한 정보를 제공합니다.

vercel.json 설정


{
  "version": 2,
  "builds": [
    {
      "src": "app.js",			
      "use": "@vercel/node"
    }
  ],
  "routes": [
    {
      "src": "/(.*)",
      "dest": "/app.js",
      "headers": {
        "Cache-Control": "no-cache, no-store, max-age=0, must-revalidate"
      }
    }
  ]
}

JSON 설정 설명

  • version: Vercel 설정 파일의 버전을 나타냅니다. 여기서는 2를 사용하고 있습니다.
  • builds: 빌드 설정을 정의합니다.
    • src: 빌드할 소스 파일을 지정합니다. 여기서는 "app.js"를 사용합니다.
    • use: 사용할 빌드 런타임을 지정합니다. "@vercel/node"는 Node.js 애플리케이션을 위한 것입니다.
  • routes: 라우팅 규칙을 정의합니다.
    • src: 모든 요청을 매칭하는 정규식입니다.
    • dest: 모든 요청을 "app.js"로 전달합니다.
    • headers: 응답 헤더를 설정합니다. 여기서는 캐시를 비활성화하는 설정을 하고 있습니다.

모든 환경과 프레임워크에 적용 가능

  • 이 캐시 설정은 Vercel을 통해 배포된 모든 정적 및 동적 파일에 적용됩니다.
  • Flask, JavaScript 등 백엔드 또는 프런트엔드 프레임워크에 상관없이 적용됩니다.
  • Vercel의 Edge Network가 요청을 처리할 때 설정한 Cache-Control 헤더가 적용됩니다.
  • 애플리케이션의 언어나 프레임워크에 구애받지 않고 캐싱 동작을 제어할 수 있습니다.

주의사항: 백엔드 서버(예: Flask)에서 직접 응답 헤더를 설정하는 경우, 해당 설정이 Vercel 설정보다 우선될 수 있습니다. 일관된 캐싱 정책을 유지하려면 Vercel 설정과 애플리케이션 코드에서 설정한 헤더 간의 충돌이 없도록 주의해야 합니다.

실제 운영 환경에서는 성능과 사용자 경험을 고려하여 적절한 캐싱 전략을 사용하여 성능을 최적화하는 것이 좋습니다. 특정 리소스나 API 엔드포인트에 대해서만 캐시를 비활성화하는 것을 고려해보세요.
