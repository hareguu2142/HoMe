Vercel Login 문제 해결

PowerShell 실행 정책 변경 오류

사용자:

PS C:\Users\isomo\OneDrive\바탕 화면\깃허브\HoMe> vercel login
vercel : 이 시스템에서 스크립트를 실행할 수 없으므로 C:\Users\isomo\AppData\Roaming\npm\vercel.ps1 파일을 로드할 수 없습니다. 자세한 내용은 about_Execution_Policies(https://go.
microsoft.com/fwlink/?LinkID=135170)를 참조하십시오.
위치 줄:1 문자:1
+ vercel login
+ ~~~~~~
    + CategoryInfo          : 보안 오류: (:) [], PSSecurityException
    + FullyQualifiedErrorId : UnauthorizedAccess

해결:

이 오류는 PowerShell에서 스크립트 실행이 차단되어 발생합니다. 이를 해결하려면 PowerShell의 실행 정책을 변경해야 합니다. 다음 단계를 따라 해보세요:

  1. PowerShell을 관리자 권한으로 실행:
    • 시작 메뉴에서 "PowerShell"을 검색하고, "Windows PowerShell"을 오른쪽 클릭한 후 "관리자 권한으로 실행"을 선택합니다.
  2. 실행 정책 변경:
    • 아래 명령어를 입력하고 Enter 키를 누릅니다:
    Set-ExecutionPolicy RemoteSigned
    • 경고 메시지가 나타나면 Y를 입력하여 변경을 승인합니다.
  3. PowerShell을 종료하고 다시 시도:
    • PowerShell을 닫은 후, 다시 일반 권한으로 PowerShell을 열고 vercel login 명령어를 실행해 보세요.

이 방법으로 문제가 해결될 것입니다!

사용자:

PS C:\Users\isomo\OneDrive\바탕 화면\깃허브\HoMe> Set-ExecutionPolicy RemoteSigned
Set-ExecutionPolicy : 레지스트리 키 'HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\PowerShell\1\ShellIds\Microsoft.PowerShell'에 대한 액세스가 거부되었습니다. 기본(LocalMachine) Scope
에 대한 실행 정책을 변경하려면 "관리자 권한으로 실행" 옵션으로 Windows PowerShell을 시작하십시오. 현재 사용자에 대한 실행 정책을 변경하려면 "Set-ExecutionPolicy -Scope CurrentU 
ser"를 실행하십시오.
위치 줄:1 문자:1
+ Set-ExecutionPolicy RemoteSigned
+ ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
    + CategoryInfo          : PermissionDenied: (:) [Set-ExecutionPolicy], UnauthorizedAccessException
    + FullyQualifiedErrorId : System.UnauthorizedAccessException,Microsoft.PowerShell.Commands.SetExecutionPolicyCommand

해결:

이 오류는 PowerShell을 관리자 권한으로 실행하지 않아서 발생합니다. 현재 사용자에 대한 실행 정책을 변경하려면 -Scope CurrentUser를 사용하여 실행하세요. 아래의 명령어를 입력해 보세요:

  1. PowerShell을 관리자 권한으로 실행 (앞서 언급한 방법으로):
    • "Windows PowerShell"을 검색한 후, 오른쪽 클릭하고 "관리자 권한으로 실행" 선택.
  2. 실행 정책 변경:
    • 아래 명령어를 입력하고 Enter 키를 누릅니다:
    Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
  3. 변경 확인:
    • 정책이 제대로 변경되었는지 확인하려면 다음 명령어를 입력하세요:
    Get-ExecutionPolicy -List
  4. PowerShell 종료 후 다시 시도:
    • 이제 일반 권한으로 PowerShell을 열고 vercel login 명령어를 다시 실행해 보세요.

이렇게 하면 문제가 해결될 것입니다!
